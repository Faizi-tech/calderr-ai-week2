# Day 3 – Tool Calling Basics with LangChain

## Overview

Day 3 focused on understanding **Tool Calling** using LangChain and Groq. Instead of relying solely on the Large Language Model's internal knowledge, custom Python functions were converted into tools that the model can identify and invoke when solving user requests.

This notebook demonstrates the creation of multiple custom tools, their individual testing, and binding them to a Large Language Model for intelligent tool selection.

---

## Objectives

- Understand the concept of Tool Calling.
- Learn how LangChain converts Python functions into tools.
- Create custom tools using the `@tool` decorator.
- Test each tool independently.
- Bind multiple tools to a Large Language Model.
- Observe how an LLM decides which tool should be used for a given query.

## Tool Calling Workflow

```
User Query
     │
     ▼
Large Language Model
     │
     ▼
Select Appropriate Tool
     │
     ▼
Generate Tool Call
     │
     ▼
Execute Tool
     │
     ▼
Return Tool Result
     │
     ▼
Final Response
```

## Key Takeaways

- Tool Calling extends the capabilities of Large Language Models.
- The LLM determines which tool should be used based on the user's request.
- Tools perform external tasks that the LLM cannot execute independently.
- Multiple tools can work together to build intelligent AI agents.
- Tool Calling forms the foundation of autonomous AI systems.
